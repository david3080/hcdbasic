# 人間中心設計基礎模擬試験

## Q1.

人間中心デザインにおいて、「デザイン」の言葉を用いる際に、留意すべき点について、最も適切なものを次の中から選びなさい。

    1.「デザイン」が色や形を創造する、狭義の意味で用いられていること。 
    2.「デザイン」がモノ・コトづくりを指す、広義の意味で用いられていること。
    3.「デザイン」がトレンドのモノを指す、先端的な意味で用いられていること。
    4.「デザイン」がユニークなアイデアに基づく、創造的な意味で用いられていること。

## A1.

2. 「デザイン」がモノ・コトづくりを指す、広義の意味で用いられていること。

説明：人間中心デザインは、人間のニーズや目標を最優先し、それを満たすために最適なソリューションを見つけるためのアプローチであり、モノだけでなくサービスやシステムなども含まれます。そのため、**「デザイン」は狭義の美的意味合いよりも、「モノ・コトづくり」を指す広義の意味で理解することが有効**です。

## Q2.

ユーザーに手順を示す説明文を書くときの方法として、最も適切なものを次の中から選びなさい。

    1. 操作と結果は一文にまとめて書く。
    2. ユーザーメリットと結果を分けて書く。
    3. 手順の目的と操作方法を一文にまとめて書く。
    4. 操作と結果の文を分けて書く。

## A2.

4. 操作と結果の文を分けて書く。

説明：操作と結果を分けて書くことで、ユーザーは手順を追いやすくなります。一文にまとめて書くと、情報が混ざり合って解読するのが難しくなる可能性があります。そのため、**各ステップが何を達成するものか（結果）とその方法（操作）を明確に区別することが重要**です。

## Q3.

専門家によるユーザビリティ評価を実施するときに、どのようなことに留意すべきか、最も適したものを次の中から選びなさい。

    1. 新たな手法や文献を参考にするよりも、評価者自身の経験を重視してユーザビリティ評価を計画することに留意する。
    2. 可能な限り、ユーザビリティ評価の裏付けとなるデータに基づく提案を行うよう留意する。
    3. ユーザビリティ評価結果は、特定の開発メンバーにだけ役立つように留意する。
    4. デザインの決め手となる部分に対して批判的な意見が出た場合は、必ずしも取り入れないなど、デザイナーの方針を優先するよう留意する。

## A3.

2. 可能な限り、ユーザビリティ評価の裏付けとなるデータに基づく提案を行うよう留意する。

説明：ユーザビリティ評価は、製品やサービスがエンドユーザーにとってどの程度使いやすいかを客観的に測定することを目指しています。そのため、評価者自身の経験や感覚だけでなく、具体的なデータに基づいた提案を行うことが重要です。これにより、評価結果の信頼性と透明性が確保され、改善点や新たな開発方針を明確にすることが可能となります。また、製品の全般的なユーザビリティを向上させるためには、評価結果を開発チーム全体で共有し、全員が参考にすることも重要です。

## Q6.

アイデアの創出を目的として行うディスカッションをする際に、留意すべきことを、次の中から選びなさい。

    1. アイデアを出す際には、あらかじめ吟味して出すようにする。
    2. 自分で考えたアイデアを重視し、他の人のアイデアの全部、あるいは一部を使って別のアイデアとして発案しない。
    3. アイデア抽出を制限するような結論を述べることは慎む。
    4. 突飛なアイデアを述べることは慎む。

## A6.

3. アイデア抽出を制限するような結論を述べることは慎む。

説明：アイデアの創出の際には、自由な発想を促進し、多様な意見やアイデアを受け入れる環境を作ることが重要です。制限するような結論を早くから述べると、他の参加者の意欲を損なったり、新しいアイデアの発生を阻害する可能性があります。一方、1と4は、アイデア創出の初期段階では制約をかけると、自由な発想が難しくなる可能性があります。2は、他の人のアイデアをベースにして新しいアイデアを発展させることも重要な方法の一つであるため、このような行為を制限することは適切ではありません。
